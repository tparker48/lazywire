im making a TUI tool called lazyscp. You start it from local location and can open 1 (or many) remote connections, which open as panes in the terminal. you press tab to switch between for navigation. You can: delete, copy, cut, paste, or rename files across any of the remotes/locals. Navigation should be easy, you can use hjkl or arrows and space to expand/collapse a folder. Or you can type the path (like you would for "cd some/path" and it will in realtime start a fuzzy file search to any path on machine. So you can navigate manually or do the fuzz serach to quickly nav on any of the machines. Once you are hovering over a folder or file, you press c,x,d,r to copy, cut, delete, rename, or p to paste inside selected folder. if file is highlihgted, p pastes to same level of selected file. under the hood it uses a combination of threading w/ ssh for exploration, and then individual scp calls for file moves across machines.
